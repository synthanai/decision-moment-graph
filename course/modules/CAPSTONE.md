# Capstone: Real Decision Project

**Time**: 30 minutes  
**Goal**: Apply everything you've learned to a real decision

---

## Welcome to the Capstone

You've learned all 8 DMG primitives:

✅ MEMO — The decision brief  
✅ MOMENT — Provenance trail  
✅ RAMP — Proportional governance  
✅ DOORS — Reversibility checklist  
✅ DISSENT — Structured disagreement  
✅ TRACE — Evidence links  
✅ COMMIT — State transitions  
✅ OUTCOME — Learning loops  

Now it's time to put them all together on a **real decision**.

---

## Choose Your Decision

Pick something meaningful. This should be:

- A decision you're **actually facing** (or recently made)
- Something with **real stakes** (not a toy example)
- Ideally **RAMP 3+** (so you practice the governance gates)

**Examples**:
- "Should we hire for Role X or promote internally?"
- "Should we adopt Tool Y or build our own?"
- "Should we prioritize Feature A or Feature B for Q2?"
- "Should we renew Vendor Z's contract?"

**My decision**: ___

## Part 1: Create the MEMO (10 min)

```markdown
# MEMO: [Your Title]

## Decision
[Clear statement of what you're committing to]

## Context

### Constraints
- [What limits you?]
- [Budget? Timeline? Resources?]

### Assumptions
- [What must be true for this to work?]
- [What are you betting on?]

## Options Considered

### Option A: [Name]
- **Description**: 
- **Pros**: 
- **Cons**: 

### Option B: [Name]
- **Description**: 
- **Pros**: 
- **Cons**: 

### Option C: [Name]
- **Description**: 
- **Pros**: 
- **Cons**: 

### Option D: Do Nothing
- **Description**: 
- **Pros**: 
- **Cons**: 

## Recommendation
[Which option and why?]

## Risks
1. [Risk 1]: Likelihood [H/M/L], Impact [H/M/L]
2. [Risk 2]: Likelihood [H/M/L], Impact [H/M/L]

## Mitigations
1. [How you'll address Risk 1]
2. [How you'll address Risk 2]

## Owners
- **Decision Owner**: [Name]
- **Reviewer**: [Name]

## Review Date
[When should this be revisited?]
```

## Part 2: Set RAMP + DOORS (5 min)

```markdown
## RAMP

**Level**: [1-5]
**Justification**: [Why this level?]
**Time to Undo**: [How long would reversal take?]

## DOORS

### D — Declare
[What exactly are we committing to?]

### O — Observe
What we'll monitor:
1. [Metric 1]
2. [Metric 2]
3. [Metric 3]

### O — Own
**Rollback Owner**: [Name + Role]

### R — Ready
[Rollback plan if needed]

### S — Signal
| Metric | Threshold | Horizon | Action |
|--------|-----------|---------|--------|
| | | | |
| | | | |
```

---

## Part 3: Capture DISSENT + TRACE (5 min)

### DISSENT

```markdown
**Author**: [Name]
**Claim**: [What they (or you) disagree with]
**Evidence**: 
1. [Supporting point]
2. [Supporting point]
**Conditions to change mind**: [What would prove this wrong?]
```

### TRACE

```markdown
**Claim**: [A key claim in your MEMO]
**Source**: [URL or reference]
**Retrieved**: [Date]
**Strength**: [strong/medium/weak]
**Quote**: [Relevant passage]
```

## Part 5: Finalize COMMIT + OUTCOME (5 min)

```markdown
## COMMIT

**State**: [Draft / Proposed / Final]
**Date**: [Today]
**By**: [Your name]

**Conditions to stay committed**:
1. [What would make you reconsider?]
2. [What would make you reconsider?]

## OUTCOME

**Next Check Date**: [When will you review this?]
```

## What Happens Next

### If this is a real decision:
1. Share the MEMO with stakeholders
2. Log events in MOMENT as things change
3. Run the OUTCOME check on the scheduled date
4. Update the verdict: keep/adjust/reverse

### To continue learning:
1. Run the capstone on 3-5 more decisions
2. Track your prediction calibration over time
3. Facilitate a team workshop using the DMG Kit
4. Contribute to the DMG open standard

You now know how to:
- Structure decisions as inspectable artifacts

## The Movement

> **Stop storing chats. Store decision objects.**

DMG makes decisions:
- Structured (not narratives)
- Auditable (not "trust me")
- Reversible (not permanent mistakes)
- Learnable (not repeated errors)

**Isolated reasoning is the enemy.**

Welcome to the future of decision governance.
