# Contributing to DMG

Thank you for your interest in contributing to the Decision Moment Graph standard!

## Ways to Contribute

### 1. Use DMG and Give Feedback
- Create MEMOs for real decisions
- Report usability issues
- Share success stories

### 2. Improve Documentation
- Fix typos and clarify language
- Add examples
- Translate materials

### 3. Contribute Code
- SDK implementations
- CLI improvements
- Conformance tests

### 4. Propose Spec Changes
- Submit a DIP (DMG Improvement Proposal)

---

## Developer Certificate of Origin (DCO)

By contributing, you agree that your contributions are your own work and you have the right to submit them under the MIT license.

Sign your commits with:

```bash
git commit -s -m "Your commit message"
```

This adds:
```
Signed-off-by: Your Name <your.email@example.com>
```

---

## Code Style

### Python
- Follow PEP 8
- Use type hints
- Run `black` and `ruff`

### JavaScript/TypeScript
- Use ESLint + Prettier
- Prefer TypeScript

### Markdown
- One sentence per line
- Use reference links for URLs

---

## Pull Request Process

1. **Fork** the repository
2. **Create a branch** from `main`
3. **Make changes** with clear commits
4. **Add tests** if applicable
5. **Update docs** if needed
6. **Submit PR** with clear description

### PR Checklist

- [ ] Code follows style guidelines
- [ ] Tests pass locally
- [ ] Documentation updated
- [ ] DCO sign-off included
- [ ] Linked to relevant issue/DIP

---

## DMG Improvement Proposals (DIPs)

For changes to the DMG Core Specification:

1. Create a new file: `community/dips/DIP-NNN_title.md`
2. Use the template in `community/dips/TEMPLATE.md`
3. Submit as a PR for discussion
4. DIPs require maintainer approval
